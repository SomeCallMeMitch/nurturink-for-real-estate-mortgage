# NurturInk Automated Campaigns V1 - Quick Start Guide for Manus Max

**Purpose:** This guide provides a quick overview for Manus Max to start implementation.

---

## What You're Building

An automated campaign system for NurturInk that sends handwritten cards on birthdays, policy start dates (welcome), and renewal dates. Insurance agents (NYL office pilot) need this to automate their client touchpoints.

## Architecture Overview

```
User creates campaign → Clients enrolled → Daily Scheduler creates ScheduledSends 
→ Approval Queue (Optional) → Send Processor executes sends → Cards mailed via Scribe API
```

**Key Components:**
- **Campaign** - Configuration (type, enrollment mode, steps, **approval toggle**)
- **CampaignStep** - Individual cards in sequence (design, message, timing)
- **CampaignEnrollment** - Who's enrolled in each campaign
- **ScheduledSend** - Pre-scheduled sends for visibility (**awaiting_approval status**)
- **Background Jobs** - Daily Scheduler (creates sends + **credit reservation**) + Send Processor (executes sends)

## Key Technical Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Timezone | Org timezone | Simpler for V1, client timezone in V2 |
| Campaign edits | Only affect future sends | Less surprising, simpler |
| Client date changes | Don't auto-reschedule | Simpler, next scheduler run picks up |
| Send time | Every 2 hours, 6am-6pm | Spreads load, allows credit purchases |
| Look-ahead window | 14 days | Good visibility, not too much stale data |
| **Approval Mode** | **Optional per campaign** | **Agents want control over high-value sends** |
| **Credit Reservation** | **14-day look-ahead** | **Prevents depletion from manual sends** |

## Critical Implementation Details

### 1. Approval Queue Logic
If `Campaign.requiresApproval` is true:
- `ScheduledSend` is created with status `awaiting_approval`.
- Send Processor skips these until status is changed to `pending` (approved).
- User approves via new `/approvals` page.

### 2. Credit Reservation Logic
- Daily Scheduler calculates total credits needed for all `ScheduledSend` records in the next 14 days.
- Credits page displays "Reserved for Campaigns" and "Available for QuickSend".
- QuickSend logic should warn if it would dip into reserved credits.

### 3. Recurring Logic
After send completes (birthday/renewal), schedule the next year's send immediately to maintain visibility in the "Reserved" total.

## Success Criteria

**User should be able to:**
1. Create a Birthday campaign in under 5 minutes.
2. **Toggle "Require Approval" for any campaign.**
3. **Review and approve cards in the Approval Queue.**
4. **See "Reserved Credits" on the Credits page.**
5. Import recipients with date fields via CSV.
6. Get notified of upcoming sends and low credits.
