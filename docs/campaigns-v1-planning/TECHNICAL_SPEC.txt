# NurturInk Automated Campaigns V1 - Technical Specification

**Version:** 1.1  
**Date:** February 1, 2026  
**Status:** Planning Phase (Updated with B44 Feedback)  
**Target:** Insurance agents (NYL office pilot)

---

## Executive Summary

This document specifies the technical implementation of Automated Campaigns for NurturInk - a feature that allows users to set up recurring, date-triggered handwritten card campaigns. The system is designed to be simple for V1 but architected for future expansion.

**Key Business Driver:** Automation increases card volume → increases revenue. Insurance agents need birthday, welcome, and renewal reminder automation.

## Architecture Overview

### Design Principles

1. **Generic Campaign Engine** - Not hardcoded for specific campaign types
2. **Template-Based** - Pre-built configurations users activate
3. **Flexible Triggers** - Support before/after date triggers
4. **Sequence Support** - 1-N steps per campaign
5. **Enrollment Rules** - Opt-in vs opt-out with exclusions
6. **Separation of Concerns** - Planning (ScheduledSend) separate from execution (actual send)
7. **Approval Workflow** - Optional "Review before send" gate for high-value touchpoints

## Data Model Specification

### New Entities

#### 1. Campaign
Defines an automated campaign configuration.

**Key Fields:**
- `orgId`, `createdBy`, `name`, `type`, `status`
- `enrollmentMode` (opt_in, opt_out)
- `triggerField` (birthday, policy_start_date, renewal_date)
- `requiresApproval` (boolean) - **NEW: If true, sends go to Approval Queue**

#### 2. CampaignStep
Defines individual steps in a campaign sequence.

**Key Fields:**
- `campaignId`, `stepOrder`, `cardDesignId`, `templateId`, `messageText`
- `timingDays`, `timingReference` (trigger_date, previous_step)

#### 3. CampaignEnrollment
Tracks which clients are enrolled in which campaigns.

**Key Fields:**
- `campaignId`, `clientId`, `status` (enrolled, excluded, completed, paused)
- `enrolledAt`, `lastSentDate`, `lastSentStep`

#### 4. ScheduledSend
Pre-scheduled card sends for visibility and processing.

**Key Fields:**
- `campaignId`, `campaignStepId`, `clientId`, `scheduledDate`
- `status` (pending, **awaiting_approval**, queued, sent, failed, skipped)
- `automationHistoryId`

## Core Features & Logic

### 1. Approval Queue (Human-in-the-loop)
Insurance agents often want to verify messages before they are sent.
- **Mechanism:** If `Campaign.requiresApproval` is true, the Send Processor marks the `ScheduledSend` as `awaiting_approval` instead of sending it.
- **UI:** A new "Approval Queue" page where users can review, edit, and approve or skip pending sends.
- **Notification:** Users receive a daily digest of cards waiting for approval.

### 2. Credit Reservation System
To prevent campaigns from failing due to unexpected credit depletion.
- **Mechanism:** When a `ScheduledSend` is created for the next 14 days, the system "earmarks" the required credits.
- **UI:** The Credits page shows "Available Balance" vs "Reserved for Campaigns".
- **Logic:** If an org tries to do a manual QuickSend that would dip into reserved credits, they receive a warning.

### 3. Refined Opt-out Logic
- **Global Opt-out:** Clients can be marked as `automation_status = 'opted_out'` to block all campaigns.
- **Campaign-specific Exclusions:** Users can exclude specific clients from specific campaigns without affecting others.
- **Auto-enrollment:** For `opt_out` campaigns, new clients matching the trigger criteria are automatically enrolled during the Daily Scheduler run.

## Background Jobs

### 1. Daily Scheduler (2:00 AM)
- Scans active campaigns.
- Identifies clients needing sends in the next 14 days.
- Creates `ScheduledSend` records.
- **NEW:** Updates Credit Reservation totals.

### 2. Send Processor (Every 2 hours, 6am-6pm)
- Finds `ScheduledSend` records for today.
- If `requiresApproval` is true and not yet approved → skip/notify.
- If approved or no approval needed → execute send via Scribe API.
- Deducts credits (from reserved pool).
- Updates history.

### 3. Notification Job (8:00 AM)
- Sends daily summary email.
- **NEW:** Includes count of cards in the Approval Queue.
- **NEW:** Warns if reserved credits exceed available balance.

## UI Specification

### 1. Campaigns List & Wizard
- Wizard includes a toggle for "Require Approval for this Campaign".
- Shows "Estimated Monthly Credits" during setup.

### 2. Approval Queue Page (`/approvals`)
- List of cards `awaiting_approval`.
- Quick "Approve All" or individual "Review & Edit".
- Search by client name.

### 3. Dashboard Widget
- Shows "Cards Sent Today", "Upcoming (7 days)", and "Awaiting Approval".
